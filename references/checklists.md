# Mission-Driven Morphogenesis Theory Checklists

## Intake Prompts

Capture these inputs before sketching or critiquing:

- Mission: what must this artifact achieve, for whom, under what conditions?
- Origin: who made it, with what technology, standards, resources, and values?
- Environment: where does it operate, and what physical or social conditions
  shape it?
- Operation: who uses it, how is it handled, maintained, supplied, stored, or
  transported?
- History: what wear, repair, retrofit, preservation, or replacement has time
  written into it?
- Constraints: what material, process, budget, time, medium, body, and safety
  limits apply?

## Mission Sentence Template

Write one sentence that includes:

1. user or owner
2. operating situation
3. target outcome
4. main constraint or risk

Example pattern:

`This is a [artifact type] used by [user] in [environment] to [outcome] while
coping with [constraint].`

## Design Workflow

Use this order when generating form:

1. Write the mission sentence.
2. Lock origin and production culture.
3. Lock environment and operation.
4. List production constraints before styling.
5. Produce 2-4 macro silhouette directions.
6. Choose the direction whose massing best fits mission and environment.
7. Place major layout elements from operation.
8. Add local structures from function.
9. Add history and surface evidence.
10. Tune readability without breaking causality.

## Macro-Form Checklist

Confirm that macro decisions answer these questions:

- Does the silhouette communicate the mission fast?
- Do proportions support the intended speed, mass, authority, fragility, or
  stability?
- Does the stance or mass distribution fit the environment?
- Do the largest volumes reveal what must be emphasized or protected?
- Would a mission swap force major changes to the outline?

## Micro-Structure Checklist

Confirm that local structures answer these questions:

- Are access points where operation demands them?
- Are vents, sensors, grips, hatches, controls, supports, or maintenance points
  present for a reason?
- Are crew, user, or operator interactions physically believable?
- Do service paths and replacement logic make sense?

## Readability Checklist

Check the artifact at three distances:

### Far Distance

- silhouette is distinct
- overall temperament is readable
- mission cue is visible

### Mid Distance

- major layout explains use
- origin and production culture become guessable
- environment logic becomes visible

### Near Distance

- joints, materials, and fastening feel plausible
- wear, repair, or preservation state feels earned
- detail density supports the main read instead of drowning it

## Constraint Checklist

Validate the design against the stated build reality:

- material logic
- manufacturing or assembly logic
- budget and time logic
- medium logic
- body and safety logic

If any item fails, revise the form before polishing.

## Evaluation Suite

Run these tests:

1. Three-second readability test
2. Ten-second origin guess test
3. Thirty-second operation guess test
4. Causal trace test
5. Subtraction test
6. Replacement test

## Output-Shaping Modes

### Concise Abstract

Use when the user wants a short theory summary.

Output blocks:

- one-line thesis
- formal model
- 3-5 key propositions

### Proposal Mode

Use when the user wants a rationale for a design direction.

Output blocks:

- mission
- origin
- environment and operation
- constraints
- macro-form choice
- micro-structure logic
- readability strategy
- risks and tradeoffs

### Creative Mecha Checklist

Use when the user wants a build sheet for a fictional machine or vehicle.

Output blocks:

- mission sentence
- origin and faction traits
- environment and operation
- silhouette and massing cues
- key local structures
- history and wear plan
- readability notes
- production compromises

### Critique Mode

Use when the user wants to pressure-test an existing design.

Output blocks:

- strongest causal decisions
- weak or decorative decisions
- readability gaps
- constraint violations
- recommended revisions

## Fast Judgement Rules

- If it reads well but has no causal backbone, treat it as decoration.
- If it is causal but unreadable, clarify silhouette, proportion, or cueing.
- If it is readable and causal but impossible to make, revise constraints first.
- If detail is doing the job of form, strengthen macro shape before adding more.
